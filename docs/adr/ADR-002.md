# ADR-002: Why YAML Frontmatter (Not JSON, TOML, or Separate Metadata)

**Decision:** YAML frontmatter in each markdown file.

**Rationale:**
- One file = one knowledge unit (no file/metadata split to get out of sync)
- YAML is more readable than JSON for humans editing files directly
- TOML lacks nested structure expressiveness for links/kpis
- Frontmatter is an established convention (Jekyll, Hugo, Obsidian, mdx)
- LLMs parse YAML frontmatter natively with high reliability
